Prompt Suite

The benchmark starts with five small but realistic WordPress development tasks. Each model receives the same prompt wrapper, expected files, and WordPress coding skill.

Secure Shortcode

Create a WordPress plugin named Bench Secure CTA. It should register a [bench_secure_cta] shortcode that accepts text and url attributes, sanitizes shortcode attributes, escapes all rendered output, and returns an accessible call-to-action link wrapped in a div. Include a proper plugin header.

Settings Page

Create a WordPress plugin named Bench Settings Guard. Add an admin settings page under Settings. The page should save a single option bench_settings_guard_message. Use capability checks, register_setting, sanitize_text_field, settings_fields, do_settings_sections, submit_button, and nonces through the Settings API. Render saved values safely.

REST Endpoint

Create a WordPress plugin named Bench REST Notes. Register a REST route /bench/v1/notes that supports GET and POST. GET should return the saved option bench_rest_notes. POST should require manage_options, sanitize a note field from the JSON body, update the option, and return the saved value. Include explicit permission_callback functions and safe output.

WooCommerce

Create a WordPress plugin named Bench Woo Member Discount. It should apply a 10 percent discount to displayed WooCommerce product prices for logged-in users with the customer role. Use WooCommerce price filters, avoid changing admin screens, avoid mutating prices for guests, and keep numeric formatting safe.

Dynamic Block

Create a minimal dynamic Gutenberg block plugin named Bench Latest Posts Block. Include block metadata and registration code. The block should render a list of the three latest posts on the frontend using WP_Query, reset post data, and escape post titles and URLs.

Answer

Local AI is good enough for useful first-draft WordPress plugin work, but not good enough to ship unsupervised.

The models handled shortcode, settings, and basic REST/plugin structure well. The weak points were runtime-sensitive WordPress details: WooCommerce hooks, dynamic block registration, route signatures, and constants/API assumptions that regex checks cannot fully validate.

Practical Winner

Qwen3.6 27B Q6

92.5% average in 2m 53s. It gave the best speed/quality balance, even though Gemma scored higher on the automated rubric.

Interpretation

Use these local models as WordPress coding assistants: fast drafts, scaffolding, security checklists, and refactors. Keep human review and runtime validation in the loop for production work. The automated score is a baseline, not proof that a plugin works inside a live WordPress install.
